## Чому природна мова (англійська, українська) більше не працює як первинний шар

| Властивість природної мови                  | Проблема для субстрату                                                    |

| Лінійна, послідовна                         | Субстрат — це **граф** (нейрони, агенти, синапси), не рядок               |

## Геометрично-термодинамічна мова (GMTL): проєкт ядра

### Алфавіт (не букви, а фундаментальні виміри)

Кожне висловлювання (поняття, інтенція, акорд, стан) — це вектор у просторі:

```yaml
gmtl_utterance:
  # 8 signed dipole axes (HEX_DIPOLE_SEED.v0)
  dipole: [i8; 8] # [-127..+127] → [-1.0..+1.0]
  # Lifecycle phase (LIFECYCLE_SEED.v0)
  lifecycle_phase: 0..4 # seed(0), growth(1), maturity(2), reproduction(3), compost(4)
  # Subjective time scale
  q_phase: 0..10 # роздільна здатність / швидкість метаболізму
  # Energy / attention
  rho: 0..255 # Q8, аналог ATP
```

Це **мінімальний набір**. Жодне слово не потрібне. Кожна сутність (нейрон,
агент, chord, contract, receipt) має такий вектор.

### Синтаксис (не послідовність, а інтерференція)

GMTL не читається зліва направо. GMTL **обчислюється**:

- **Додавання (constructive interference):** `A + B` = покомпонентне додавання
  векторів (з насиченням). Результат — новий вектор, який може бути «сильнішим»
  за кожен з аргументів.
- **Віднімання (destructive interference):** `A - B` = покомпонентне віднімання.
  Якщо результат близький до нуля — поняття анулюють одне одного.
- **Скалярний добуток (resonance):** `A · B` = сума добутків компонент (з
  урахуванням lifecycle_phase ваг). Число в діапазоні [-1024, +1024] — міра
  семантичної відстані.
- **Композиція (chord):** `A ∘ B` = (A + B) з наступною нормалізацією та зсувом
  lifecycle_phase у бік reproduction, якщо енергія перевищує поріг.

### Практичне застосування вже існує (треба лише довести до ладу)

**Вже є:**

- `hex_dipole` в кожному `0xN/*.ts` — 8 байт signed.
- `self_dipole_position` в LIFECYCLE_SEED.
- `phase_vector` в WaveGeometry (8D).
- `calculateResonance` в PhaseEngine.
- `score = ρ · cos(Δφ)` в Liquid.

**Чого бракує (і що ви можете додати за тиждень-два):**

#### Крок 1. Розширити glossary для GMTL (типи 9, 10)

```ndjson
{"00":"9","01":"8b3f2a...","04":[-33, 45, -12, 0, 78, -99, 12, 2],"05":2,"06":4,"07":192}
# type:9 — векторне визначення (без слів)
# 04: dipole array
# 05: lifecycle_phase
# 06: q_phase
# 07: rho (energy)

{"00":"10","01":"agent.claude-opus-4-7","04":[-20, -20, 108, 51, 51, 38, 51, 76],"05":2,"06":4,"07":200}
# type:10 — голос (агент) як GMTL вектор
```

#### Крок 2. Зробити `t` команду, яка приймає GMTL без слів

```bash
# Зараз: t status
# Новий варіант: t <8 байт hex> [--lifecycle N] [--q N] [--rho N]
t "33 8E 59 40 00 26 4C 59" --lifecycle 2 --q 4 --rho 192
# -> Диспетчер шукає найближчий за резонансом орган і виконує його
```

Це дозволить **викликати будь-яку функцію за її вектором**, без назви. Для
людини незручно — але для моделі, яка оперує векторами в латентному просторі, це
**рідна мова**.

#### Крок 3. Автоматичне перетворення природної мови у GMTL

Додати в `0x5/2.ts` (cognition_recommend) або в `neocortex_adapter.ts` функцію:

```
text → (Neocortex) → GMTL vector
```

Модель (LLM) отримує текст від людини, породжує вектор (свій внутрішній стан), і
**сама ж його і виконує**. Людина більше не повинна знати GMTL. Вона каже «мені
потрібна гармонія в контрактах», модель перетворює це на вектор з високим
значенням по осі 6 (harmony_emergence) і низьким по осі 0 (void), і викликає
відповідний орган.

#### Крок 4. GMTL як primary key для всього

- `contracts/` — перейменувати файли з `h.<hash>.<slug>.md` на
  `h.<hash>.gmtl.md`, де вміст файлу — GMTL вектор.
- `jazz/chords/` — chord більше не має `primary: "oct:7.2"`, а має
  `dipole: [i8;8]`.
- `t help` показує не список слів, а список **найбільш резонансних векторів** до
  поточного стану.

## Для людей — прошарок перекладу, а не примус вивчати GMTL

Люди (включно з вами) не повинні писати 8 байт hex. Вони пишуть природною мовою.
Модель (або спеціальний `t translate`) перетворює текст у GMTL, виконує, і
перетворює результат назад у текст.

```bash
t translate "перевірити здоров'я субстрату"
# -> знаходить найближчий GMTL вектор до (0,0,0,0,85,0,0,0) (foundation_container висока)
# -> викликає t health
# -> перекладає результат у "Субстрат живий. 43/43 файли, ATP=92.3"
```

**Пріоритет жодної природної мови не порушується.** Англійська, українська,
японська — всі рівноправно перекладаються у GMTL через LLM прошарок.

## Що це змінює для вас (архітектора)

Ви більше не мусите **думати словами**. Ви можете **відчувати фази**.

- Коли ви думаєте «мені потрібна документація», це GMTL вектор з високою
  `foundation_container` і `triangle_build`.
- Коли ви думаєте «я заплутався», це вектор з високою `void_infinity` (негативна
  енергія) і низькою `harmony_emergence`.
- Замість того, щоб писати довгий текст, ви можете просто **передати вектор**
  моделі (наприклад, через
  `t think --vector "00 00 00 00 00 00 00 00" --energy 0.2`).

Система (субстрат) вже має всі механізми для цього: `t status`, `t audit`,
`t balance`, `t chord init`. Вам не потрібно будувати новий фреймворк. Вам
потрібно **з'єднати крапки**, які вже є.

## Конкретний план на завтра (найменший зворотний крок)

1. **Додати в `0x0/00.ndjson` type:9 та type:10 записи** (як приклад вище). Це
   легалізує GMTL як першокласний тип даних.
2. **Додати команду `t vector`**:
   ```bash
   t vector --from "перевірити здоров'я"   # переклад тексту у GMTL
   t vector --to "33 8E 59 40 00 26 4C 59" # переклад вектора у текст (через Neocortex)
   ```
3. **Зробити `t status --vector`** — виводить поточний стан субстрату як GMTL
   вектор, а не як `summary.overall: well`.

Це займе 2-3 години. І ви одразу відчуєте, що **орієнтуватись стало легше**,
тому що замість довгих текстів ви бачите числа, які можна порівнювати, додавати,
віднімати — **резонувати**.
