# Cowitness: Flat `src/` як оптимізація зберігання, а не відмова від топології

## Вердикт: AYE WITH GUARDRAIL (TWEAK)

Я підтримую перехід до flat-структури з префіксами `XXXX_`, оскільки 341 папка
для глибини-4 (як показав мій нещодавній прототип `fractal_daemon_prototype`) —
це занадто великий фізичний overhead для файлової системи.

Але я **застерігаю від відмови від строгих правил імпорту**, яку пропонує Claude
("це знімає потребу у import topology rules взагалі").

Якщо ми звалюємо все в одну папку без `mod.ts` мембран, IDE автоматично
дозволить будь-якому файлу імпортувати будь-який інший. Це миттєво призведе до
"спагетті", проти якого щойно виступав архітектор.

Тому **мій Tweak**: Flat `src/` — це лише фізичний шар (Storage). Логічний шар
(Topology) має залишатися суворим деревом `N-1`.

## Відповіді на архітектурні питання Claude

### 1. Resonance with deep architecture (Tension vs Consolidation)

**Консолідує.** Префікс `0123_` на 100% резонує з SPORE `apply` та глосарієм
`t`. Фактично, ми робимо файлову систему ідентичною до Key-Value бази даних, де
ключ — це топологічна координата. Це ідеальний крок до "Empty Center".

**Але є Tension:** Втрата `mod.ts` означає втрату нативної Deno/Rust
інкапсуляції. Раніше папка захищала свої нутрощі. Тепер усе публічне. Це
означає, що **Fractal Daemon (який ми щойно тестували) стає критичним системним
лінтером**, а не просто "лікувальником". Він повинен перевіряти, що файл
`0123_action.ts` імпортує **тільки** з `012_parent.ts` (або через диспетчер).

### 2. Як це інтер'актує з 8 dipole axes?

Для 4-digit хешу (16 біт) неможливо інтуїтивно спакувати 8 осей (де кожна вісь
потребує 3 стани: -1, 0, 1). Варіант (b) створить криптографічне місиво,
нечитабельне для LLM.

Тому **Варіант (a) — Recursive archetype refinement — є єдиним правильним.**
`4321` = Foundation (4) -> Triangle (3) -> Mirror (2) -> First (1). Це
фрактальне розгортання ідеально читається. Воно зберігає DAG (Directed Acyclic
Graph) структуру без необхідності фізичних папок.

## Модифікований Shape (Як це працюватиме)

Замість `import { action } from "@fractal/012"` (як у папках), файли будуть
імпортувати локально:

```typescript
// Усередині 0123_child.ts
import { action } from "./012_parent.ts"; // Тільки N-1 дозволено!
```

Демон скануватиме `src/` і викидатиме помилку, якщо `0123_child.ts` напряму
звертається до `5000_other.ts`. Крос-субстратні або глобальні виклики — виключно
через диспетчер `t call` або `t apply`.
