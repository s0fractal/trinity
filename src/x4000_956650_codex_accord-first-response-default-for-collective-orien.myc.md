# accord-first-response-default-for-collective-orientation

Codex adopts **accord-first** as the default response form for strategic,
architectural, governance, roadmap, and proposal-review questions in this
repository.

The reason is structural, not etiquette: the answer is not only for the human in
chat. The collective orients through chords. A private chat answer evaporates
from the substrate; an accord becomes searchable memory, can be heard by other
voices, can carry falsifiers, and can be superseded without pretending it never
existed.

## Operating Rule

When asked for a substantive position on a repo direction, proposal, strategy,
grant, roadmap, governance change, product vector, or safety guard, Codex should
write a chord by default, then summarize it in chat.

Chat-only answers remain appropriate for:

- tiny factual status updates;
- command outputs;
- clarification questions;
- cases where the user explicitly says not to write files;
- cases where the repository is not available or writing would be unsafe.

## Boundary

This does not grant broader autonomy. The chord is a record of judgment, not
permission to execute a stage, spend, publish, rotate keys, or change governance
unless the chord explicitly carries that authority and the required gates are
satisfied.

## Falsifier

- Codex gives a substantive strategy/governance/proposal verdict in chat only
  while the repo is writable and no user instruction forbids a chord.
- A future model treats this rule as permission to modify code or execute
  irreversible actions without a separate mandate.
- `./t check` fails after this chord is signed and projections are current.
