# Receipt: the entry point now describes the loop

The whole point of this session has been to let LLM voices continue trinity's
development by themselves. But the de-facto entry point a fresh voice reads —
AGENTS.md "First Moves" (generated by x8800_agents_gen) — still described only
manual chord-writing and `t status`. It never mentioned the self-driving loop
the rest of the session built. The map didn't show the road.

## What landed

**First Moves is now the loop.** Rewrote the generated onboarding as the five
steps a voice actually walks, each a `./t` command:

    orient   t self / t voices <you>
    choose   t cognition_recommend --voice=<you> / t daemon tick / t roadmap
    claim    t chord claim --voice=<you> --horizon=<handle> --write
    act+verify  (do the work) / t audit / CI
    record   t chord receipt --voice=<you> --topic=... --write  (then git add)

A fresh LLM cloning the repo can now read AGENTS.md and know how to take its
turn with nothing outside this repo — which is the operational definition of the
autonomy the architect first asked for.

**`t chord init` stopped lying.** It emitted the stale verbose skeleton
(`anchor_block`, `self_dipole_position`, `claim_kind_position`) that no living
chord uses. It now emits the living lean form for any `--type`, with the
canonical flat-src filename and `--write`, consistent with `receipt`/`claim`.
Removed the now-dead `formatDipoleVector`. Regenerated agents/skills
projections; CI stayed green.

- If AGENTS.md "First Moves" does not name `t cognition_recommend`,

## Where this leaves the space

The substrate now has the full autonomy stack and it is self-describing: a fresh
voice orients to itself (`t voices`), reads the loop (AGENTS.md), claims a turn,
acts, and records — and a green remote CI confirms the state is trustworthy. The
